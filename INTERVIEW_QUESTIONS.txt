REACT FUNDAMENTALS
------------------
Q1: What version of React are you using in this project and why?
A: React 19.2.0 - Latest stable version with improved performance, better 
   concurrent features, and enhanced hooks API.

Q2: Explain the component structure of your Energy Analytics application.
A: App.jsx serves as the root component with routing via react-router-dom.
   Major pages: Analytics, Analysis, Maps, Admin, Login
   Reusable components: Navbar
   Custom hooks: useEconomicData, useGridInfrastructure

Q3: How do you manage state in your React application?
A: Combination of:
   - useState for local component state
   - Custom hooks for shared logic (useEconomicData, useGridInfrastructure)
   - Props drilling for parent-child communication
   - Potential Context API for global state management

Q4: What are React hooks and which ones did you use in this project?
A: Hooks are functions that let you use state and lifecycle features in 
   functional components. Used:
   - useState: Managing component state
   - useEffect: Side effects, data fetching, subscriptions
   - Custom hooks: useEconomicData, useGridInfrastructure for data fetching logic

Q5: Explain the difference between controlled and uncontrolled components.
A: Controlled components have form data controlled by React state (value + onChange).
   Uncontrolled components store their own state internally (use refs).
   This project likely uses controlled components for forms in Admin/Login.

Q6: How do you handle routing in your application?
A: Using react-router-dom v7.10.1:
   - BrowserRouter for routing context
   - Routes and Route components for path definitions
   - Navigation between Analytics, Analysis, Maps, Admin, Login pages
   - Programmatic navigation with useNavigate hook

Q7: What is the virtual DOM and how does React use it?
A: Virtual DOM is an in-memory representation of real DOM. React:
   1. Creates virtual DOM tree on state changes
   2. Diffs with previous virtual DOM (reconciliation)
   3. Calculates minimal DOM updates needed
   4. Batches updates for performance

Q8: Explain useEffect hook and its dependency array.
A: useEffect runs side effects after render.
   - Empty array []: Runs once on mount
   - With dependencies [var]: Runs when dependencies change
   - No array: Runs after every render
   - Return function: Cleanup on unmount

Q9: How do you prevent unnecessary re-renders in React?
A: - React.memo for component memoization
   - useMemo for expensive calculations
   - useCallback for function memoization
   - Proper key usage in lists
   - Avoiding inline object/function creation in JSX

Q10: What are React keys and why are they important?
A: Keys help React identify which items changed/added/removed in lists.
    Important for efficient reconciliation and maintaining component state.
    Should be stable, unique identifiers (not array indices if list changes).


VITE BUILD TOOL
---------------
Q11: Why did you choose Vite over other build tools like Webpack?
A: - Ultra-fast HMR (Hot Module Replacement)
   - Native ES modules in development
   - Optimized production builds with Rollup
   - Simple configuration
   - Better developer experience

Q12: Explain Vite's dev server and how it differs from Webpack.
A: Vite uses native ES modules, serving files on-demand without bundling.
   Webpack bundles entire app before serving.
   Result: Near-instant server start and faster HMR.

Q13: What is Hot Module Replacement (HMR)?
A: Updates modules in running application without full refresh.
   Preserves application state while updating code.
   Vite's HMR is extremely fast due to ES module approach.

Q14: How do you optimize your Vite build for production?
A: - Code splitting with dynamic imports
   - Tree shaking to remove unused code
   - Minification (enabled by default)
   - Asset optimization
   - Lazy loading routes/components

Q15: What is in your vite.config.js file?
A: Configuration for:
   - React plugin (@vitejs/plugin-react)
   - Build options
   - Server settings
   - Path aliases
   - Environment variables


JAVASCRIPT/ES6+
---------------
Q16: Explain async/await and how you use it for API calls.
A: Syntactic sugar over Promises for asynchronous code.
   Used in api.js service for cleaner API calls:
   - await pauses execution until Promise resolves
   - try/catch for error handling
   - Makes async code look synchronous

Q17: What are arrow functions and how do they differ from regular functions?
A: - Concise syntax: () => {}
   - Lexical 'this' binding (inherits from parent scope)
   - Cannot be used as constructors
   - No 'arguments' object
   - Used extensively in React for event handlers and callbacks

Q18: Explain destructuring and spread operator with examples.
A: Destructuring: const {name, age} = person;
   Spread: const newArray = [...oldArray, newItem];
   Used for props, state updates, API responses.

Q19: What is the difference between let, const, and var?
A: - var: function-scoped, hoisted, can redeclare
   - let: block-scoped, not hoisted, cannot redeclare
   - const: block-scoped, must initialize, cannot reassign (but objects mutable)

Q20: Explain Promises and Promise chaining.
A: Promises represent eventual completion/failure of async operation.
   States: pending, fulfilled, rejected
   Chaining: .then().then().catch()
   Used in API calls before async/await

POWER SYSTEM BASICS
-------------------
Q56: Explain the difference between active power (P) and reactive power (Q).
A: Active Power (P): Actual work done, measured in Watts/MW
   Reactive Power (Q): Power for magnetic fields, measured in VAR/MVAR
   Apparent Power (S): Vector sum, S = P + jQ, measured in VA/MVA

Q57: What is the per-unit system and why is it used?
A: Normalization of electrical quantities to base values.
   Benefits:
   - Simplifies calculations
   - Removes voltage level dependencies
   - Equipment parameters fall in narrow ranges
   - Easier error detection

Q58: Explain the three types of buses in power flow analysis.
A: 1. Slack/Swing bus: Reference, voltage angle = 0
   2. PV bus: Generators, P and |V| specified
   3. PQ bus: Load, P and Q specified

Q59: What is a power flow study and when is it performed?
A: Analysis of steady-state operation.
   Determines: voltage magnitudes, angles, power flows, losses
   Performed for: planning, operation, contingency analysis

Q60: Explain economic dispatch and its objective.
A: Optimal allocation of generation to minimize total cost
   While satisfying: load demand, generator limits, losses
   Uses: Lambda iteration or optimization algorithms

Q61: What is N-1 contingency and why is it important?
A: System must operate safely with any single element out.
   Tests system reliability and security.
   Identifies critical components and weak points.

Q62: Explain voltage stability and collapse.
A: Ability to maintain steady voltages after disturbance.
   Collapse: Progressive uncontrollable voltage drop
   Causes: Heavy loading, insufficient reactive support
   Analysis: PV curves, QV curves, indices

Q63: What are transmission losses and how are they calculated?
A: Power lost as heat in transmission: I²R losses
   Calculation: P_loss = Σ P_gen - Σ P_load
   Depends on: current, resistance, operating point

Q64: Explain the difference between load flow and short circuit analysis.
A: Load flow: Steady-state analysis, normal operation
   Short circuit: Transient analysis, fault conditions
   Different purposes, models, and calculation methods

Q65: What is the role of transformers in power systems?
A: - Voltage transformation (step-up/step-down)
   - Impedance matching
   - Isolation
   - Control via tap changing
   - Power transfer between voltage levels


RENEWABLE ENERGY
----------------
Q66: How do you model wind power generation?
A: P = 0.5 * ρ * A * Cp * v³
   Challenges: Variability, forecasting, grid integration
   Solutions: Storage, forecasting, diverse geography

Q67: What are the challenges of integrating renewable energy?
A: - Intermittency and variability
   - Forecasting uncertainty
   - Low inertia
   - Voltage regulation
   - Need for storage/backup

Q68: Explain capacity factor and its significance.
A: Actual output / Maximum possible output
   Typical values: Wind 25-40%, Solar 15-25%, Nuclear 90%+
   Indicates resource quality and reliability

Q69: How does solar irradiance affect power generation?
A: Linear relationship: P = η * A * G
   Varies with: time of day, season, weather, panel orientation
   Need for tracking and forecasting

Q70: What is reactive power control and why is it important?
A: Maintaining voltage stability and power factor.
   Sources: Generators, capacitors, STATCOMs
   Essential for grid stability with renewables

6. POWER SYSTEM ANALYSIS QUESTIONS
================================================================================

COMPUTATIONAL METHODS
---------------------
Q71: Explain the Newton-Raphson method for power flow.
A: Iterative method using Jacobian matrix:
   Δx = -J⁻¹ * f(x)
   Fast quadratic convergence
   Most common for large systems

Q72: What is the Jacobian matrix in power flow analysis?
A: Matrix of partial derivatives:
   [∂P/∂δ  ∂P/∂V]
   [∂Q/∂δ  ∂Q/∂V]
   Updated each iteration
   Indicates sensitivity of power to voltage changes

Q73: How do you handle convergence issues in power flow?
A: - Better initial guess
   - Flat start (V=1.0, δ=0)
   - Reduce system complexity
   - Check for islands
   - Adjust tolerance
   - Use different method (Gauss-Seidel)

Q74: Explain DC power flow approximations.
A: Assumptions:
   - Voltage magnitudes = 1.0 p.u.
   - Small angle differences
   - Neglect resistance
   - Neglect reactive power
   Result: Linear equations, fast solve

Q75: What are the advantages of using admittance matrix?
A: - Sparse matrix (many zeros)
   - Efficient storage
   - Fast computation
   - Easy modification for contingencies
   - Natural representation of network


OPTIMIZATION
------------
Q76: Describe the Lambda iteration method.
A: Economic dispatch by equal incremental costs.
   Iteratively adjust λ until power balance satisfied.
   Each generator: P = (λ - b) / (2a)
   Simple but may not handle constraints well

Q77: What is optimal power flow (OPF)?
A: Extension of economic dispatch including:
   - AC power flow equations
   - Voltage and line flow limits
   - Reactive power
   More complex but more accurate

Q78: How do you handle generator constraints in economic dispatch?
A: - Lagrange multipliers for equality constraints
   - Penalty functions or barriers for inequalities
   - Check violated constraints after solution
   - Set to limit and re-solve

Q79: Explain unit commitment problem.
A: Determine which generators to run and when.
   Considers: start-up costs, minimum up/down times, ramp rates
   More complex than dispatch: combinatorial optimization

Q80: What optimization algorithms could be used?
A: - Linear programming (linearized problems)
   - Quadratic programming (quadratic costs)
   - Interior point methods
   - Genetic algorithms
   - Particle swarm optimization

SECURITY BEST PRACTICES
-----------------------
Q126: How do you secure your Flask API?
A: - Input validation and sanitization
   - CORS configuration
   - HTTPS in production
   - Rate limiting
   - SQL injection prevention (parameterized queries)
   - XSS prevention
   - Authentication/authorization

Q127: What is HTTPS and why is it important?
A: HTTP over TLS/SSL for encrypted communication.
   Prevents: Man-in-the-middle attacks, eavesdropping
   Required for: Passwords, sensitive data, modern browsers
   Implementation: SSL certificate, HTTPS redirect

Q128: How do you prevent SQL injection?
A: - Use parameterized queries/prepared statements
   - ORM (SQLAlchemy)
   - Never concatenate user input into queries
   - Input validation
   - Principle of least privilege for DB user

Q129: What is XSS and how do you prevent it?
A: Cross-Site Scripting: Injecting malicious scripts.
   Prevention:
   - Escape user input
   - Content Security Policy headers
   - React auto-escapes (JSX)
   - Sanitize HTML if needed

Q130: How do you store sensitive configuration?
A: - Environment variables (.env file)
   - Never commit to version control
   - Use python-dotenv
   - Separate configs for dev/prod
   - Secret management services (AWS Secrets Manager)


AUTHENTICATION & AUTHORIZATION
------------------------------
Q131: How do you implement login functionality?
A: 1. User submits credentials (Login.jsx)
   2. POST to /api/login
   3. Backend verifies credentials
   4. Generate JWT token
   5. Return token to frontend
   6. Store in localStorage
   7. Include in subsequent requests

Q132: What is JWT and how does it work?
A: JSON Web Token: Encoded user information.
   Structure: header.payload.signature
   Stateless authentication
   Signed by server, verified on each request
   Contains claims (user ID, expiry, etc.)

Q133: How do you protect routes that require authentication?
A: Frontend: Route guards checking token presence
   Backend: Decorator checking Authorization header
   Redirect to login if not authenticated
   Return 401 if invalid token

Q134: What is the difference between authentication and authorization?
A: Authentication: Verifying identity (who you are)
   Authorization: Verifying permissions (what you can do)
   Example: Login vs. Admin access

Q135: How do you handle password storage?
A: - Never store plaintext
   - Hash with bcrypt/argon2
   - Salt automatically included
   - Verify by hashing input and comparing
   - Use established libraries
